## Documentation Standards

All documentation follows these principles:

### 1. **Clarity Over Brevity**
We prefer precise, detailed explanations over terse summaries. Technical accuracy is paramount.

### 2. **Progressive Disclosure**
- High-level overview → Detailed explanations → Implementation specifics
- Each document can be read independently

### 3. **Code Examples**
Every concept illustrated with runnable code snippets.

### 4. **Audience-Specific**
Each document clearly states its target audience and assumed knowledge.

### 5. **Maintainability**
- Cross-references between documents
- Version information included
- Update history tracked
